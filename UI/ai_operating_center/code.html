Authentication API

The Knower OS utilizes OAuth 2.0 for secure access to enterprise endpoints. All requests must include a valid bearer token in the authorization header.

auto_awesome AI Generated Example 
curl -X POST https://api.knower.os/v1/auth \
  -H "Content-Type: application/json" \
  -d '{"client_id": "YOUR_CLIENT_ID", "client_secret": "YOUR_SECRET"}'

Rate Limits

Standard tier accounts are limited to 1000 requests per minute. Enterprise accounts have custom limits defined in their SLA.

// Reference: auth_service.ts
export async function generateToken(
  clientId: string, 
  secret: string
) {
  const client = await db.clients.find(clientId);
  
  if (!client || !verifyHash(secret, client.hash)) {
    throw new AuthError('Invalid credentials');
  }

  // AI Note: Token expiry logic was recently 
  // updated to 24h for enterprise tier.
  return jwt.sign({ sub: clientId }, env.JWT_SECRET, { 
    expiresIn: client.tier === 'enterprise' ? '24h' : '1h' 
  });
}
